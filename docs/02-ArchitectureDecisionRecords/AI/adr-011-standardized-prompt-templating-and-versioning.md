# ADR 011: Standardized Prompt Templating and Versioning

- **Date**: 2026-05-10
- **Status**: 📝 Draft
- **Deciders**: Architecture Team
- **Technical Story**: VK.Blocks.AI / Prompting

## 1. Context (背景)

AI アプリケーションの性能はプロンプトの質に大きく依存します。しかし、プロンプトを C# のコード内に文字列としてハードコードすると、メンテナンス性、柔軟性、および非エンジニア（プロンプトエンジニア等）との協業において大きな障害となります。

## 2. Problem Statement (問題定義)

1. **更新サイクルの不一致**: プロンプトの微調整のために、アプリケーション全体の再コンパイルとデプロイが必要になる。
2. **可読性の低下**: コード内に巨大な文字列が混在し、ビジネスロジックの見通しが悪くなる。
3. **ローカライズの困難さ**: 言語ごとにプロンプトを切り替える仕組みを個別に実装する必要がある。
4. **テスト・比較の欠如**: どのバージョンのプロンプトが現在の出力をもたらしたのかを追跡できない。

## 3. Decision (決定事項)

1. **テンプレートエンジン（Liquid / Handlebars）の採用**:
   プロンプトを単なる文字列ではなく、動的な変数を埋め込み可能なテンプレートとして扱います。VK.Blocks では原則として `Liquid` を推奨とします。

2. **`IVKPromptProvider` 抽象化の導入**:
   プロンプトのロード元を隠蔽するインターフェースを定義します。
   - `EmbeddedPromptProvider`: アセンブリ内のリソースから読み込む（初期開発用）。
   - `FilePromptProvider`: ローカルファイルから読み込む。
   - `BlobPromptProvider`: クラウドストレージ（S3/Azure Blob）から読み込む。

3. **プロンプトの「リソース化」**:
   プロンプトは「ロジック」ではなく「データ/設定」として扱い、デプロイなしでの更新を可能にします。

4. **`VKPromptRegistry` によるバージョン管理**:
   各プロンプトに一意の ID とバージョンを付与し、実行時に特定のバージョンを指定（または A/B テスト）できる仕組みを提供します。

## 4. Alternatives Considered (代替案の検討)

- **Option 1: C# 11 の Raw String Literals を使う**
  - **Rejected Reason**: 短期的な可読性は改善するが、デプロイなしでの更新や非エンジニアとの協業という課題は解決しない。
- **Option 2: Microsoft Semantic Kernel の `.prompty` 形式を直接使う**
  - **Rejected Reason**: 独自のメタデータ（VK.Blocks 固有の認可情報等）を付与し、プロバイダー非依存を維持するため、ラップした形式を採用する。

## 5. Consequences & Mitigation (結果と緩和策)

- **Positive**: 
  - プロンプトの改善をアプリケーションのデプロイなしに即時反映できる。
  - プロンプトエンジニアが GitHub の Markdown ファイルなどを編集するだけで AI の挙動を調整できる。
- **Negative**:
  - ファイル I/O やクラウド通信が発生するため、プロンプトのロードにわずかなオーバーヘッドが生じる。
- **Mitigation**:
  - キャッシュ機構（InMemory / Distributed Cache）を標準で統合し、パフォーマンスへの影響を最小限に抑えます。

## 6. Implementation & Security (実装詳細 and セキュリティ考察)

- **実装詳細**: プロンプトファイルには YAML フロントマター形式を採用し、モデルのパラメータ（Temperature 等）のデフォルト値をプロンプト側に定義できるようにします。
- **セキュリティ考察**: テンプレートのレンダリングにおいて「Prompt Injection」を受けないよう、入力変数のサニタイズを徹底します。また、プロンプトテンプレート自体が改ざんされないよう、ロード元のストレージに対するアクセス権限を厳格に管理します。
